Server¶

Run duruses-server to allow clients to access a set of databases stored in a directory.

Prepare a directory for data storage¶

Create a directory to store Durus database files.

Consider using the .duruses extension to help you remember that it contains Durus database files.

$ mkdir appdata.duruses

Starting a server¶

Run duruses-server, giving it the name of the directory to serve from. The program will respond with the host and port it is listening on:

$ duruses-server appdata.duruses
Listening on 127.0.0.1:22972

Override the host and/or port by using --host and --port. For example, you can listen on all network interfaces at port 3333:

$ duruses-server --host=0.0.0.0 --port=3333 appdata.duruses
Listening on 0.0.0.0:3333

Quiet log output by setting --loglevel:

$ duruses-server --loglevel=100

Stopping a server¶

Press Control-C where duruses-server is running. Duruses will disconnect all connected clients and stop listening for new connections.
